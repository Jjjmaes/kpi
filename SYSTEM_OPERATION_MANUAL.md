## 系统概述

### 系统简介

语家 OA 系统是一个专为翻译公司设计的办公自动化系统，支持项目全生命周期管理、自动化 KPI 计算、财务管理、行政办公和邮件通知等功能。

### 主要功能模块

- **用户管理**：用户创建、角色分配、权限管理
- **客户管理**：客户信息管理、联系人管理
- **项目管理**：项目创建、成员分配、状态跟踪、附件上传
- **KPI 管理**：自动计算绩效、月度汇总、审核
- **财务管理**：发票申请、回款管理、财务统计
- **邮件通知**：项目分配邮件、附件发送

### 系统角色

| 角色 | 权限说明 |
|------|---------|
| **管理员 (admin)** | 所有权限，可管理用户、配置系统 |
| **财务 (finance)** | 查看所有项目、管理发票和回款 |
| **销售 (sales)** | 创建项目、管理自己创建的客户和项目 |
| **兼职销售 (part_time_sales)** | 创建项目、管理自己创建的客户和项目 |
| **项目经理 (pm)** | 查看分配给自己的项目、管理项目成员 |
| **翻译 (translator)** | 查看分配给自己的项目 |
| **审校 (reviewer)** | 查看分配给自己的项目 |
| **排版 (layout)** | 查看分配给自己的项目 |
| **综合岗 (admin_staff)** | 查看所有项目、管理项目 |

## 登录与用户管理

### 1. 系统登录

#### 操作步骤

1. 打开系统登录页面
2. 输入用户名和密码
3. 点击"登录"按钮
4. 首次登录或密码重置后，系统会提示修改密码

#### 注意事项

- 密码复杂度要求：
  - 至少 8 位
  - 包含大写字母、小写字母、数字和特殊字符
- 如果忘记密码，请联系管理员重置

### 2. 用户管理（仅管理员）

#### 2.1 创建用户

**操作步骤：**

1. 进入"用户管理"页面
2. 点击"创建用户"按钮
3. 填写用户信息：
   - **姓名** *（必填）
   - **用户名** *（必填，唯一）
   - **邮箱** *（必填，唯一）
   - **电话**（可选）
   - **角色** *（至少选择一个）
   - **专/兼职**（默认：专职）
   - **状态**（默认：激活）
4. 点击"创建"按钮

**注意事项：**

- 用户名和邮箱必须唯一
- 创建后系统会生成随机密码，管理员可以复制密码告知用户
- 用户首次登录需要修改密码

#### 2.2 编辑用户

**操作步骤：**

1. 在用户列表中找到要编辑的用户
2. 点击"编辑"按钮
3. 修改用户信息（用户名不可修改）
4. 点击"更新"按钮

**可修改字段：**

- 姓名
- 邮箱
- 电话
- 角色
- 状态（激活/禁用）
- 专/兼职

#### 2.3 重置密码

**操作步骤：**

1. 在用户列表中找到要重置密码的用户
2. 点击"重置密码"按钮
3. 系统会生成新密码并显示
4. 复制密码告知用户

#### 2.4 删除用户

**操作步骤：**

1. 在用户列表中找到要删除的用户
2. 点击"删除"按钮
3. 确认删除操作

**注意：** 删除用户实际上是软删除（设置为禁用状态），不会真正删除数据。

### 3. 个人中心

#### 3.1 修改个人信息

**操作步骤：**

1. 点击右上角用户名，选择"个人中心"
2. 在"基本信息"卡片中修改：
   - **邮箱** *（必填）
   - **电话**（可选）
3. 点击"更新信息"按钮

**注意：** 用户名和姓名只能由管理员修改。

#### 3.2 修改密码

**操作步骤：**

1. 在个人中心页面
2. 在"修改密码"卡片中填写：
   - **当前密码** *
   - **新密码** *（需符合复杂度要求）
   - **确认新密码** *
3. 点击"修改密码"按钮

## 项目管理

### 1. 创建项目（销售、兼职销售、管理员）

#### 操作步骤

1. 进入"项目管理"页面
2. 点击"创建项目"按钮
3. 填写项目基本信息：

   **基本信息：**
   - **项目编号**（可选，留空自动生成）
   - **项目名称** *（必填）
   - **选择客户** *（必填）
   - **选择联系人**（可选，从客户联系人中选择）
   - **业务类型** *（笔译/口译/转录/本地化/其他）
   - **项目类型**（笔译项目：MTPE/深度编辑/审校项目/混合类型）

   **语种信息：**
   - **源语种** *（必填）
   - **目标语种** *（至少一个，可添加多个）

   **金额信息：**
   - **字数**（笔译项目）
   - **单价**（每千字或每小时）
   - **项目总金额** *（必填）
   - **含税**（勾选表示金额含税）
   - **需要发票**（勾选表示需要开具发票）

   **时间信息：**
   - **交付时间** *（必填）
   - **预计回款时间**（默认3个月后）

   **特殊要求：**
   - 术语表要求
   - NDA 要求
   - 参考文件
   - 纯翻译交付
   - 双语交付
   - 打印盖章并快递
   - 备注

4. **添加项目成员**（可选）：
   - 选择角色（项目经理/翻译/审校/排版等）
   - 选择用户
   - 对于翻译角色，需要选择翻译类型（MTPE/深度编辑）和字数占比
   - 对于排版角色：
     - **专职排版**：可以设置排版费用（不超过项目金额的5%），费用用于记录，走KPI计算
     - **兼职排版**：必须输入兼职排版费用（不超过项目金额的5%），费用直接使用，不走KPI计算
   - 对于其他兼职角色（除兼职销售外）：
     - 必须输入兼职费用（如兼职翻译费用）
     - 费用直接使用，不走KPI计算
   - 点击"添加成员"按钮

5. **上传项目附件**（可选）：
   - 点击文件选择框
   - 选择要上传的文件（可多选）
   - 文件总大小不超过 20MB
   - 附件会通过邮件发送给项目成员，不会保存到服务器

6. 点击"创建项目"按钮

#### 权限说明

- **销售/兼职销售**：只能创建项目，只能查看和编辑自己创建的项目
- **管理员**：可以创建、查看和编辑所有项目
- **项目经理**：只能查看分配给自己的项目
- **翻译/审校/排版**：只能查看分配给自己的项目

### 2. 查看项目列表

#### 筛选功能

- **按月份筛选**：选择项目创建月份
- **按状态筛选**：待开始/待安排/进行中/已完成/已取消
- **按业务类型筛选**：笔译/口译/转录/本地化/其他
- **按角色筛选**：查看特定角色的项目（仅多角色用户显示）
- **按客户筛选**：选择特定客户的项目
- **按开票状态筛选**：未申请/待审批/已开票/已拒绝
- **按回款状态筛选**：未支付/部分支付/已支付
- **回款预警快捷筛选**：在业务看板点击"回款预警/近7天回款预警"卡片会跳转至项目列表，并仅显示预警项目，便于导出或后续操作
- **金额可见性**：仅管理员/财务/销售/兼职销售可见金额，综合岗不再显示项目金额

#### 角色筛选说明

- **多角色用户**：如果用户拥有多个项目成员相关角色（如翻译和审校），会显示角色筛选器
- **自动过滤**：在顶部切换角色时，项目列表会自动根据当前角色过滤
- **手动筛选**：也可以在项目列表的角色筛选器中选择特定角色或"全部角色"
- **单角色用户**：如果用户只有一个角色，角色筛选器会自动隐藏

#### 项目状态说明

- **待开始**：项目已创建，未开始
- **待安排**：项目已创建，等待安排成员或等待成员确认
- **进行中**：项目已开始，正在执行（所有生产人员都已接受）
- **已完成**：项目已完成
- **已取消**：项目已取消

### 5. 项目成员管理

#### 5.1 添加成员

**在创建项目时：**
- 在项目创建表单中添加成员

**在编辑项目时：**
1. 打开项目编辑表单
2. 在"项目成员"区域点击"添加成员"按钮
3. 选择角色和用户
4. 填写相关信息：
   - **翻译角色**：选择翻译类型（MTPE/深度编辑）和字数占比
   - **兼职排版**：必须输入兼职排版费用（不超过项目金额的5%）
   - **专职排版**：可以设置排版费用（不超过项目金额的5%）
   - **其他兼职角色**（除兼职销售外）：必须输入兼职费用
5. 点击"添加"按钮

**费用输入说明：**
- 系统会根据选择的用户自动判断是否为兼职
- 如果是兼职用户且角色需要费用输入，会自动显示费用输入字段
- 兼职排版费用不能超过项目总金额的5%
- 其他兼职角色费用不能超过项目总金额

**成员接受状态：**
- **生产人员**（非管理角色）：添加后状态为"待确认"，需要成员主动接受
- **管理人员**（管理角色，如项目经理、销售等）：添加后自动接受，无需确认
- **判断标准**：系统根据角色的`isManagementRole`字段自动判断，新创建的角色如果设置了该字段，会自动应用相应逻辑

#### 5.2 成员接受/拒绝项目分配

**操作步骤（生产人员）：**

1. **查看项目详情**：被添加的生产人员登录后，在项目详情页面可以看到自己的成员记录
2. **查看状态**：如果状态为"⏳ 待确认"，会显示"接受"和"拒绝"按钮
3. **接受项目**：
   - 点击"接受"按钮
   - 状态变为"✅ 已接受"，显示接受日期
   - 项目经理和项目创建者会收到通知
4. **拒绝项目**：
   - 点击"拒绝"按钮
   - 输入拒绝原因（可选）
   - 状态变为"❌ 已拒绝"，显示拒绝日期和原因
   - 项目经理和项目创建者会收到通知

**项目状态影响：**
- 当所有生产人员都接受后，项目状态自动变为"进行中"
- 如果有成员拒绝，项目保持"待安排"状态，需要重新安排人员
- 拒绝后重新安排的新成员需要重新确认

#### 5.3 删除成员

**操作步骤：**

1. 在项目编辑表单的"项目成员"区域
2. 找到要删除的成员
3. 点击"删除"按钮
4. 确认删除

**注意：** 只有管理员、项目经理和项目创建人可以管理成员。

### 6. 项目执行与交付流程

#### 6.1 项目执行阶段（生产 → PM）

**参与角色：翻译 / 审校 / 排版 / 项目经理**

- **生产人员标记完成（阶段性交付给 PM）**
  - 在项目详情页面的"项目管理"区域：
    - 翻译角色：点击 **"翻译完成"**
    - 审校角色：点击 **"审校完成"**
    - 排版角色：点击 **"排版完成"**
  - 系统会弹出 **"阶段性交付" 弹窗**：
    - **交付说明**（可选）：填写当前版本说明、注意事项等
    - **交付附件**（可选，可多选，建议总大小 ≤ 20MB）：上传翻译件、对齐文件等
  - 提交后系统行为：
    - 更新项目状态为对应阶段：`translation_done` / `review_done` / `layout_done`
    - 如果上传了附件：系统会给项目经理（PM）发送阶段性交付邮件，并附上所有附件
    - 如果未上传附件：只更新状态，不发邮件

> 阶段性交付是 **生产人员 → 项目经理** 的内部交付，不直接面向客户。

#### 6.2 项目经理内部交付（PM → 销售）

**参与角色：项目经理（pm）**

- **入口：**
  - 使用 PM 角色登录
  - 打开项目详情，在"项目管理"区域可以看到 **"提交给销售"** 按钮（仅 PM 角色可见）

- **操作步骤：**
  1. 点击 **"提交给销售"**
  2. 在弹出的 **"提交给销售（内部交付）"** 弹窗中填写：
     - **提交说明**（可选）：对本次交付的说明，如合并版本、质量说明等
     - **提交附件**（可选，可多选，建议总大小 ≤ 20MB）：最终整合好的内部交付包
  3. 点击 **"发送给销售"**

- **系统行为：**
  - 通过邮件将说明和附件发送给 **项目创建人（销售 / 兼职销售）**
  - **不会改变项目状态**（仍然可能是 `in_progress` / `translation_done` 等）
  - **不会生成 KPI 记录**

> 这是 **PM → 销售** 的内部交付动作，用于把整理好的交付包交给销售做客户沟通准备。

#### 6.3 最终交付给客户（销售 → 客户）

**参与角色：销售 / 兼职销售 / 管理员**

- **入口：**
  - 使用销售 / 兼职销售 / 管理员角色登录
  - 在项目详情的"项目管理"区域中，当项目状态为 `in_progress` / `translation_done` / `review_done` / `layout_done` 时，会显示 **"交付项目"** 按钮

- **操作步骤：**
  1. 点击 **"交付项目"**
  2. 在 **"项目最终交付"** 弹窗中：
     - **最终交付说明**（可选）：给内部记录或客户的简要说明
     - **最终交付附件**（可选，可多选，建议总大小 ≤ 20MB）：准备发送给客户的最终文件（同时也会发给销售自己留存）
  3. 点击 **"确认项目完成并发送"**

- **系统行为：**
  - 校验项目是否满足完成条件：
    - 已分配成员且所有生产人员已接受项目
    - 项目金额已填写且有效
  - 将项目状态更新为 **"已完成（completed）"**，记录完成时间，触发 KPI 计算
  - 如上传了最终交付附件：
    - 通过邮件将说明和附件发送给 **项目创建人（销售）**，用于留档或转发给客户
  - 向相关成员发送项目完成通知（站内通知）

> 只有 **"交付项目"** 才表示对客户的最终交付，并驱动 KPI 结算和项目关闭；
> 阶段性交付和 PM 提交给销售，都是内部流转，不会把项目标记为已完成。

### 7. 项目状态操作

#### 6.1 项目状态自动流转

**状态流转规则：**

1. **待开始** → **待安排**：
   - 添加生产人员后自动变为"待安排"
   - 项目等待成员确认

2. **待安排** → **进行中**：
   - 当所有生产人员都接受项目分配后，自动变为"进行中"
   - 如果某个角色有拒绝的成员，需要重新安排后才能进入"进行中"

3. **进行中** → **已完成**：
   - 手动点击"完成项目"按钮
   - 系统会检查所有生产人员是否都已接受，如果有拒绝的成员，无法完成项目

**注意事项：**
- 如果成员拒绝项目，项目状态会保持"待安排"，直到重新安排并接受
- 每个生产角色（翻译、审校、排版等）都需要有有效的（非拒绝的）成员且都已接受
- 拒绝的成员记录会保留，但不会影响项目状态判断（如果该角色已有新的有效成员）

#### 6.2 开始项目（已废弃）

**注意：** 此功能已废弃，项目状态现在根据成员接受情况自动流转。

#### 7.1 完成项目（旧逻辑说明，已被“交付项目”替代）

**操作步骤：**

1. 在项目详情页面
2. 点击"完成项目"按钮
3. 系统会检查：
   - 所有生产人员是否都已接受
   - 如果有拒绝的成员，需要先重新安排
4. 如果检查通过，项目状态变为"已完成"

#### 7.2 标记返修

**操作步骤：**

1. 在项目详情页面
2. 点击"标记返修"按钮
3. 填写返修原因
4. 确认标记

#### 7.3 标记延期

**操作步骤：**

1. 在项目详情页面
2. 点击"标记延期"按钮
3. 填写延期原因和新的交付时间
4. 确认标记

#### 7.4 标记客诉

**操作步骤：**

1. 在项目详情页面
2. 点击"标记客诉"按钮
3. 填写客诉原因
4. 确认标记

### 7. 导出报价单

#### 操作步骤

1. 在项目详情页面
2. 点击"导出报价单"按钮
3. 系统会生成 Excel 格式的报价单并下载

**权限说明：**

- 项目创建人（销售/兼职销售）可以导出
- 管理员和财务可以导出
- 如果项目创建人同时是项目经理，也可以导出

   - 在"KPI系数"列输入系数值（0-1之间）
   - 系数会在项目创建时自动锁定到项目中
4. 点击"更新配置"按钮保存

**新角色说明：**
- 只有满足以下条件的角色才会显示在配置列表中：
  - 不是系统角色（`isSystem !== true`）
  - 不是固定角色（`isFixedRole !== true`）
  - 不是特殊角色（`isSpecialRole !== true`）
  - 可用于KPI记录（`canBeKpiRole === true`）
- 创建新角色并设置"可用于KPI记录"后，会自动显示在配置列表中

**注意事项：**
- 系数变更会影响新创建的项目
- 已创建的项目使用锁定系数，不受影响
- 系统会记录配置变更历史
- 固定角色的系数在上面已配置，新角色的系数在下方配置

### 2. 语言管理（管理员）

#### 添加语言

1. 进入"系统配置"页面
2. 在"语言管理"区域点击"添加语言"
3. 填写语言信息：
   - **语言名称** *（必填）
   - **语言代码**（可选，如 en, zh）
   - **本地名称**（可选）
   - **状态**（激活/禁用）
4. 点击"创建"按钮

#### 编辑语言

1. 在语言列表中找到要编辑的语言
2. 点击"编辑"按钮
3. 修改语言信息
4. 点击"更新"按钮

### 3. 角色管理（管理员）

#### 3.1 查看角色

1. 进入"系统配置"页面
2. 在"角色管理"区域查看所有角色
3. 查看角色权限配置和属性设置

#### 3.2 创建角色

**操作步骤：**

1. 在"角色管理"区域点击"创建角色"按钮
2. 填写角色信息：
   - **角色代码** *（必填，只能包含小写字母、数字和下划线，且必须以字母开头）
   - **角色名称** *（必填）
   - **描述**（可选）
   - **优先级**（数字，用于排序）
   - **是否启用**（默认：启用）
   - **是否系统角色**（系统角色不能删除）
   - **是否可用于项目成员**（默认：是）
   - **是否可用于KPI记录**（默认：是）
   - **是否为管理角色**（管理角色添加为项目成员时自动接受，无需确认）
   - **是否为固定角色**（固定角色的KPI系数在KPI配置页面中配置）
   - **是否为特殊角色**（有特殊处理逻辑的角色）
   - **是否可记录产能**（如翻译、审校等需要记录工作量的角色）
   - **是否可作为评价人**（如pm、translator、reviewer、layout等）
   - **是否可被评价**（如sales、part_time_sales、pm等）
3. 配置权限（选择每个权限的值）
4. 点击"创建"按钮

**注意事项：**
- 角色代码创建后不可修改
- 系统角色不能删除，但可以禁用
- 管理角色添加为项目成员时自动接受，无需成员确认
- 生产角色（非管理角色）添加为项目成员时需要成员确认

#### 3.3 编辑角色

**操作步骤：**

1. 在角色列表中找到要编辑的角色
2. 点击"编辑"按钮
3. 修改角色信息（角色代码不可修改）
4. 修改权限配置
5. 点击"更新"按钮

**可修改字段：**
- 角色名称
- 描述
- 优先级
- 是否启用
- 是否可用于项目成员
- 是否可用于KPI记录
- 是否为管理角色
- 是否为固定角色
- 是否为特殊角色
- 是否可记录产能
- 是否可作为评价人
- 是否可被评价
- 权限配置

#### 3.4 角色属性说明

**角色属性字段：**

- **isManagementRole（是否为管理角色）**：
  - `true`：管理角色，添加为项目成员时自动接受，无需确认
  - `false`：生产角色，添加为项目成员时需要成员确认
  - 示例：pm、sales、admin_staff、finance等为管理角色

- **isFixedRole（是否为固定角色）**：
  - `true`：固定角色，KPI系数在KPI配置页面的固定角色区域配置
  - `false`：新角色，KPI系数在KPI配置页面的新角色区域配置
  - 示例：translator、reviewer、pm、sales、admin_staff、finance为固定角色

- **isSpecialRole（是否为特殊角色）**：
  - `true`：特殊角色，有特殊处理逻辑（如part_time_sales、part_time_translator、layout）
  - `false`：普通角色，使用标准KPI计算逻辑

- **canRecordCapacity（是否可记录产能）**：
  - `true`：可以记录产能（如翻译、审校等需要记录工作量的角色）
  - `false`：不记录产能

- **canBeEvaluator（是否可作为评价人）**：
  - `true`：可以作为评价人（如pm、translator、reviewer、layout等）
  - `false`：不能作为评价人

- **canBeEvaluated（是否可被评价）**：
  - `true`：可以被评价（如sales、part_time_sales、pm等）
  - `false`：不能被评价

#### 3.5 角色权限说明

- **project.view**：项目查看权限（all/assigned/sales/self）
- **project.edit**：项目编辑权限
- **project.create**：项目创建权限
- **project.delete**：项目删除权限
- **project.member.manage**：项目成员管理权限
- **kpi.view**：KPI 查看权限（all/self）
- **kpi.view.self**：查看自己的KPI权限
- **kpi.config**：KPI配置权限
- **finance.view**：财务查看权限
- **finance.edit**：财务编辑权限
- **customer.view**：客户查看权限（all/self）
- **customer.edit**：客户编辑权限（all/self）
- **user.manage**：用户管理权限
- **system.config**：系统配置权限
- **role.manage**：角色管理权限

## 评价与反馈

### 1. 评价类型与权限

- 项目经理 → 销售：PM 可针对项目的销售提交评价
- 翻译/审校/排版 → 项目经理：执行角色可评价该项目的 PM
- 评价仅用于反馈，不影响 KPI 计算
- 支持匿名提交（如前端提供匿名选项）

### 2. 提交评价

1. 在项目详情页点击评价入口（针对销售或项目经理）
2. 选择评价对象并填写评分、文字反馈，可选匿名
3. 提交后记录在项目评价列表

### 3. 查看评价

- 在项目详情页查看该项目的评价列表
- 在个人中心/用户详情查看“我的评价统计”（平均分与近期评价）

### 4. 时间限制

- 仅已完成的项目可评价
- 默认完成后 30 天内允许评价，超期不可提交

## 常见问题

### 1. 登录问题

**Q: 忘记密码怎么办？**

A: 请联系管理员重置密码。

**Q: 首次登录提示修改密码？**

A: 这是正常的安全设置，请按照密码复杂度要求设置新密码。

### 2. 权限问题

**Q: 为什么看不到某些项目？**

A: 根据角色权限，您只能查看：
- 自己创建的项目（销售/兼职销售）
- 分配给自己的项目（项目经理/翻译/审校/排版）
- 所有项目（管理员/财务）

**Q: 为什么不能编辑项目？**

A: 只有管理员和项目创建人可以编辑项目。

### 3. 项目问题

**Q: 如何添加项目成员？**

A: 在创建项目时添加，或在项目编辑页面添加。

**Q: 项目附件在哪里？**

A: 附件通过邮件发送给项目成员，不会保存在系统中。

**Q: 如何导出报价单？**

A: 在项目详情页面点击"导出报价单"按钮。

### 4. KPI 问题

**Q: KPI 如何计算？**

A: 系统根据项目金额、字数、系数自动计算。

**Q: 如何生成月度 KPI？**

A: 管理员或财务在 KPI 管理页面选择月份后点击"生成月度 KPI"。

### 5. 财务问题

**Q: 如何申请开票？**

A: 销售/兼职销售在财务管理页面点击"申请开票"，选择项目并填写信息。

**Q: 如何添加回款记录？**

A: 管理员或财务在财务管理页面点击"添加回款"，选择项目并填写回款信息。

### 6. 邮件问题

**Q: 为什么没有收到项目分配邮件？**

A: 检查以下几点：
1. 邮件服务是否已配置（RESEND_API_KEY, RESEND_FROM_EMAIL）
2. 用户邮箱是否正确
3. 查看服务器日志确认发送状态
4. 检查垃圾邮件文件夹

**Q: 附件大小限制是多少？**

A: 单个文件不超过 20MB，所有附件总大小不超过 20MB。

**Q: 报销申请的发票照片在哪里？**

A: 发票照片通过邮件发送给财务和管理员，不会保存在系统中。请检查邮箱中的报销申请通知邮件。

### 7. 系统配置问题

**Q: 如何修改 KPI 系数？**

A: 管理员在系统配置页面修改，注意已创建的项目不受影响。

**Q: 如何添加新语言？**

A: 管理员在系统配置页面的语言管理区域添加。
